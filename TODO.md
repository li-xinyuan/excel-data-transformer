# TODO - 项目改进清单

> 创建时间: 2026-03-24
> 最后更新: 2026-03-24

## 高优先级 (High Priority)

### 1. 用户体验
- [ ] **增加loading状态提示** - 文件分析、预览等操作无加载状态，用户不知道系统在处理中
- [ ] **大文件处理优化** - Excel一次性读入内存，大文件可能导致内存问题或浏览器崩溃
- [ ] **完善错误提示** - 错误提示过于技术化，对普通用户不友好

### 2. 安全性
- [ ] **XSS防护** - 用户输入（配置名称、字段名等）未做严格过滤
- [ ] **前端文件大小限制** - base64传输文件无大小限制

## 中优先级 (Medium Priority)

### 3. 功能完整性
- [ ] **拖拽上传** - 目前只支持点击上传，不支持拖拽
- [ ] **边界情况处理** - 空行/空值/特殊字符处理不完善
- [ ] **文件解析校验** - 缺少对Excel结构完整性的校验

### 4. 技术架构
- [ ] **代码拆分** - `utils.js` 和 `server.js` 过大，建议按功能拆分
- [ ] **代码重复** - `validateFileType`/`validateFileSize` 多处重复，可抽取为公共函数
- [ ] **模块耦合** - 业务逻辑与会话管理耦合严重
- [ ] **统一错误处理** - 异步操作缺少统一的错误捕获和响应格式

### 5. 性能
- [ ] **会话清理** - 长时间运行的服务器未自动清理过期会话，可能内存泄漏
- [ ] **会话存储优化** - 当前会话存储在cookie中，大数据量时可能性能问题

### 6. 安全性
- [ ] **表达式注入风险** - `expr-eval` 解析逻辑表达式可能存在注入风险

## 低优先级 (Low Priority)

### 7. 性能优化
- [ ] **映射算法效率** - 大数据量时字段映射算法效率较低
- [ ] **数组遍历优化** - 部分地方使用了不够高效的数组遍历方法

### 8. 功能增强
- [ ] **上传进度条** - 文件上传无进度反馈
- [ ] **断点续传** - 不支持大文件断点续传
- [ ] **缓存策略** - 配置文件缓存策略可优化

### 9. 其他
- [ ] **单元测试** - 缺少单元测试，代码稳定性难以保证
- [ ] **日志规范** - 日志级别和格式可进一步规范化
