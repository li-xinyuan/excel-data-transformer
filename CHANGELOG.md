# 更新日志

## [2.1.0] - 2026-04-05

### 🆕 核心新功能：数据筛选（预览弹窗集成版）

- **筛选面板嵌入预览弹窗**：数据筛选功能从主页面完整迁移至预览模态框，点击"需要对数据进行筛选"展开侧边栏面板
- **筛选条件管理**：
  - 支持多条件组合（AND / OR 模式）
  - 条件类型：等于、不等于、大于、小于、包含、正则匹配等
  - 动态增删改筛选规则，实时预览筛选结果
- **"只显示筛选后的数据"切换**：勾选后仅展示满足条件的行，自动跳转到第一个有效行
- **筛选状态指示**：
  - 筛选面板触发按钮显示红色角标（显示条件数量）
  - 被过滤的行以灰色删除线样式标识
  - 筛选后行数统计实时更新（如 "筛选后行数：956 (原始: 1,000 行)"）

### 🧭 行号导航器全面重构

- **智能定位（findNearestValidRow）**：
  - 输入不满足筛选条件的行号时，自动向后/向前搜索最近的有效行
  - 例：输入 `1`（第1行被过滤）→ 自动跳到第3行（第一个满足条件的行）
  - 输入已满足条件的行号 → 直接跳转，无额外开销
- **按钮布局优化**：上下箭头改为**左右排列**，红色问题箭头统一放在右侧
  ```
   [▲] [▲]    ← 左:上一行 | 右:上一个问题(红)
   [ 23 ]
   [▼] [▼]    ← 左:下一行 | 右:下一个问题(红)
  ```
- **边界状态禁用**：
  - 在最小行/最小有效行时 → 两个向上箭头变灰不可点
  - 在最大行/最大有效行时 → 两个向下箭头变灰不可点
  - 无问题行时 → 红色箭头始终变灰
- **问题行导航不再循环**：修复之前在边界处会跳到另一端的问题

### 🔢 预览行数显示统一

- **单一数据源原则**："预览行数"的 `/` 前数字**始终从行号选择器 input 元素读取**，保证两个数字100%同步、同时刷新
- **总行数固定显示源数据原始行数**：`/` 后面的数字固定为 1000（总行数），不受"只显示筛选后的数据"勾选影响
- **影响范围**：`gotoRow()`、`changeRow()`、`gotoProblemRow()`、`toggleFilteredView()` 全部统一调用链

### 🔗 连线显示逻辑优化

- **被过滤行不绘制连线**：当当前行是被过滤的行（灰色删除线显示），源字段与目标字段之间的绿色映射连线不再显示
  - 语义：被过滤的行不会被映射到目标字段
- **修复索引系统不一致Bug**：解决筛选模式下连线误判问题（之前用原始数组索引访问过滤后数组导致取错行）

### 💾 配置变动追踪系统

- **配置脏标记（configDirty）**：
  - 新增/修改/删除映射关系 → 按钮开始蓝色脉冲闪烁
  - 修改默认值、值转换规则、逻辑规则 → 同上
  - 修改筛选条件（增/删/改/启用/禁用/组合方式）→ 同上
- **智能对比恢复**：
  - 增加一条连线后又删除 → 配置与保存前一致 → 按钮**自动停止闪烁**
  - 基于配置快照 JSON 序列化对比，非简单布尔标记
- **覆盖全部19个变更场景**：映射(5)、默认值(2)、值转换(4)、逻辑规则(2)、筛选条件(5)、文件重置(1)
- **双按钮联动**：主界面"💾 保存配置" + 预览弹窗"💾 保存"按钮同步闪烁状态

### 🛠️ 代码质量提升

- **DEBUG 日志开关**（[app.js](src/public/js/app.js)）：
  - 新增 `APP_DEBUG` 常量（生产环境 `false`）
  - 所有 33 处 `console.log/warn` 统一替换为 `log()/warn()`
  - 需要调试时改一个常量即可全局开启
- **CSS 类提取**（[style.css](src/public/css/style.css)）：
  - 新增 **22 个语义化 CSS 类**，替代 ~25 处超长内联 style 字符串
  - 行号选择器：`.row-nav-*` 系列（8个类）
  - 弹窗通用：`.dlg-*` 系列（14个类）
  - 支持尺寸变体（`dlg-box--sm` / `dlg-box--lg`）、状态变体（`dlg-hint--warn` / `dlg-hint--info`）

### 🐛 问题修复

- **修复 changeRow() 调用顺序错误**：先渲染再同步显示，解决点击下一行时"预览行数"不更新的问题
- **修复 gotoProblemRow() 循环跳转**：移除循环逻辑，到达边界时不再跳到另一端
- **修复 findNearestValidRow 索引混乱**：返回过滤后数组索引而非原始数组索引，解决输入2却跳到23的问题
- **修复 drawConnections 取错行**：从 DOM 判断当前行是否被过滤，而非用 currentRowIndex 访问原始数组
- **修复 saveFilterToConfig 404 错误**：改用 localStorage 保存（服务端无 /api/config/save 接口）
- **修复 updatePreviewStatistics 直接赋值**：不再绕过统一的 updatePreviewRowDisplay()
